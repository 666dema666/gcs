GURPS Character Sheet Release Notes

Version 3.1.0.13 — January 31, 2007

Features

  • There is now a preference (in the "Sheet" tab) that controls whether the standard method of calculating dice of damage is used, or the optional method from B269 is used.
  • Feature #1635546: Encumbrance not reflected in skills.
  • Skills can now provide weapons.
  • Feature #1640629: Weaponmaster damage. Added the ability to specify weapon damage bonuses as a feature attached to an item. Note that the Weaponmaster advantage, if taken, will still need to be updated to reflect just which weapons it grants damage bonuses to.
  • Added an option to specify the HTML template file to use when generating HTML from the command line.
  • Added a preference to control which HTML template file is used by default.
  • Partial implementation of Feature #1641005: Support for enhancements and limitations. This feature is partially implemented in that you can now add enhancements and limitations to (dis)advantages... however there still needs to be support for creating stock versions and a way to easily select from them for application to a (dis)advantage. Thanks to Sebastien Wilhelmi for initial contributions in this area!

Bug Fixes

  • Bug #1648935: DR drops to 1. This was actually a more fundamental screw-up on my part... when prepping the code for the initial open-source release, I somehow managed to mangle the leveled bonus class, which, in turn, mangled the data files.
  • Bug #1635547: Techniques do not drag-and-drop right.
  • Bug #1642308: Miscalculated defaults.
  • Bug #1646281: Items "slide off" weapon list.
  • Bug #1643782: Damage bonus due to high skill when kicking. Actually fixed by virtue of adding Feature #1640629.

Data Files

  • Due to Bug #1648935 reverted the data files to their state as of the 3.1.0.11 release, then tried to reinstate changes made since then.
  • Updated the Boxing, Brawling, Karate, Judo, Climbing, Swimming & Stealth skills to use the new capabilities.
  • Fixed various mathematics and navigation skill prerequisites. Contributed by Zachary St. Lawrence.

Miscellaneous

  • Updated URLs used for the mailing lists to use the new SourceForge site.
  • Added menu items to take users to the bug and feature trackers on SourceForge.
  • Updated the version checker to use the new SourceForge site.
  • Eliminated the "todo" docs, as I've created entries for them in the SourceForge tracker system.

Version 3.1.0.12 — January 2, 2007

Features

  • Initial open-source release. GURPS Character Sheet is now licensed under the MPL (http://www.mozilla.org/MPL/).

Bug Fixes

  • Move can now be lowered to zero, but not by encumbrance (B17 states that encumbrance can't lower it below 1). This allows a character with no legs to be created correctly.
  • Minimum strength of weapons is now taken into account when calculating the skill level with that weapon.

Data Files

  • Reworked the "Scroll" spell prerequisites to include native language as a possibility.
  • Fixed the prerequisites on the "Teleport" spell.
  • Removed the documentation that had previously been contributed as I was getting feedback that it wasn't useful.

Version 3.1.0.7 — April 2, 2006

Features

  • Added the ability specify skill specializations when specifying skill bonuses. Updated the four 'Absolute Direction' advantages, as well as 'Talent (Musical Ability)' accordingly.
  • Added the Block stat to Melee Weapons, and updated the various weapons, shields & cloaks appropriately. If you want an item to display a Block stat, you just need to give it a melee weapon description (you can leave the actual weapon fields empty -- see 'Force Shield' as an example).
  • Unspent points now automatically adjust as a character is modified. Changed the title 'Unspent' to 'Earned' in the points display to reflect that these represent earned but unspent points. This change means that a character's point total (which now includes the 'Earned' points, unlike in the past) won't change unless the user changes the 'Earned' points -- everything else just adjusts the 'Earned' points so that the total remains the same.
  • Integer input fields now respond to the increment/decrement menu items.
  • Implemented a minimalistic XML parser to drastically reduce the RAM requirements during file loads.
  • Added a preference for controlling whether font anti-aliasing is used or not. Some systems (particularly those with older graphics cards) have drastically worse text rendering performance when this was turned on, so I've provided a way for you to turn it off if it is affecting you. Keep in mind that the smaller font sizes may be unreadable on the screen without this enabled, but they should still print fine.
  • Added menu items for jumping to the GCS web site, as well as for jumping to the GCS mailing lists web site.

Bug Fixes

  • Fixed a problem which prevented the cost reduction feature from being restored from a saved file.
  • Attempted to fix an issue on some platforms where a window could sometimes come to the foreground but not be the active window.
  • Fixed some issues that were preventing the display from updating to display changes made via features.
  • Fixed issue where case mattered in feature application.
  • Fixed some more focus issues.

Data Files

  • Added the 'Infinity Equipment' equipment list, contributed by Scott Ellsworth.
  • Various minor tweaks to correct list data, sent in from various users.

Version 3.1.0.6 — March 5, 2006

Features

  • Spells can now have weapon descriptions attached to them, just like equipment and advantages.
  • Added automatic update checking. The program will only notify you once for each update newer than the one you're running. There is also a menu item (under the Help menu) that shows the current status -- and, in the case where there is an update, will take you to the web site if you select it.
  • The storage format for preferences has been changed to XML. A side effect of this is that your old preferences will be discarded and you'll have to set them again.

Data File Updates

  • Updated all spells with weapon statistics, as needed. Contributed by Al Cardenas.

Version 3.1.0.5 — February 26, 2006

Features

In addition, options were added to override the paper size and margins for the generated files. For those of you running on a Mac, the actual executable is buried down inside the application bundle, so if you want to use this feature, you'll probably want to make a soft link or shell alias to it for convenience.

Version 3.1.0.4 — February 19, 2006

Features

  • Added the ability to modify the command keys associated with menu items. Check out the "Menu Keys" tab in preferences...
  • Added direct PDF creation via the "Save As..." menu item.
  • Separated the melee & ranged weapon stats and gave each their own stat line in the weapon list. In addition, weapons that had more than one mode of operation (for example, a sword can be thrust or swung) now list the stats separately. For weapons like a Knife, whose skill values differ depending upon the mode of operation (melee or thrown), you can now specify separate defaults. Old weapon entries are converted as much as possible — you should carefully review any existing weapons on your character sheet and adjust them as needed.

Bug Fixes

  • Restored the automatic resolution of Damage, Range & Parry in the weapon list. When I moved to the new list format, I ended up displaying the raw value rather than the resolved one, which wasn't my intention.

Version 3.1.0.3 — February 5, 2006

Features

  • Added another string comparison type, "is anything", to allow a particular field to always match.
  • Added support for modifying the Size Modifier via a Feature. Updated the Dwarfism and Gigantism (dis)advantages appropriately, as well as the Dwarf racial template.
  • Advantage containers can now act in one of three ways:
    1. Group (works as it used to, and is the default)
    2. Meta-Trait (acts as one normal trait, listed as an advantage if its point total is positive, or a disadvantage if it is negative)
    3. Race (point cost tracked separately from normal advantages and disadvantages)
  • Added support for specifying a tech level for a spell. Updated the various Technology College spells that required a TL.
  • The total carried weight is now shown in parentheses next to the "Carried Equipment" title.

Bug Fixes

  • Altering basic lift (via Strength, for example) will now cause the current encumbrance level highlight to change, if needed.
  • Used the new "is anything" match type to fix the inability to correctly match when the notes field wasn't important.
  • The title of the weapons section is now titled "Weapons" rather than the generic "Description".
  • The "equipped" state of equipment no longer affects your character's encumbrance or whether the item will show up in the weapon list. It now only affects whether features of the equipment (such as a ring that gives +1 to ST) are applied to the character, as originally intended.

Version 3.1.0.1 — January 22, 2006

Features

  • The nameable feature (where you can embed @foo@ in a name to have the user prompted to type some text to replace it when it is placed into a sheet) now extends into most (if not all) text fields of (Dis)Advantages, Skills, Spells & Equipment. In addition, if you use the same name in more than one place, it will only be prompted for once and substituted into all occurrences.
  • Removed the ability to directly edit fields within the lists — you must now bring up the detailed editor to make changes. The detailed editors are now capable of editing all information for the item.
  • Provided a separate field for specializations.
  • Provided a separate field for notes for all list types.
  • New column layout for most items.
  • Added a first pass at a user's manual by Brad Rice. Please be sure to give him some feedback.

Bug Fixes

  • While I haven't experienced this myself, some users were reporting either a failure to load saved sheets, or the sheet being huge (as if the paper were dozens of feet across). The only thing in common with these folks seemed to be inkjet printers (although I may have missed a laser printer in a bug report). Anyway, I've added code that is much more careful about trying to restore printer settings that the printer originally reported as being able to support — I suspect these printers were reporting some feature as being available, but would then error out when I tried to restore that setting. Should this happen in this release, the code ignores the error and proceeds with the next setting. This should prevent the huge/no paper size from occuring, as I don't think it was ever getting that far before failing. Those of you experiencing the problem, please send me a quick email to let me know if this fixes your problem, as I have no way to test for it.
  • When a file failed to save, there was generally no feedback to the user. Now, if a file fails to save for any reason, a dialog box will be put up to warn the user.

Miscellaneous

  • Firearms, High-Tech & Ultra-Tech Armor added, courtesy of Karl Gunnarsson.

Version 3.0.0.25 — January 2, 2006

Features

  • When launching the program, you now get a file dialog so that you can choose to open an existing file, or create a new window.
  • Implemented a complete replacement for the Sun page setup and print dialogs, along with all the logic that is associated with them. What this means for the end user is that all settings (that I can get to, which I think are most of them) are equally available on all platforms and presented to the end user in the same way. In addition, it allowed me to fix a whole host of bugs that Sun's three(!) different implementations had. As a side-effect, I've removed the preference that would show the minimum printer margins on the displayed character sheet -- there was no reliable way to get this data from within Java on anything other than Windows, so I didn't think it was worth keeping. For those that liked it the way it was, you can still use the native page setup/print dialogs by choosing to do so in preferences. If you do so, be aware that most of the print settings in the native dialogs cannot be saved and restored, so you'll have to manually set them up every time you want to print, just like in previous versions of GCS.
  • Spells in templates now allow points to be specified.

Bug Fixes

  • Fixed an infinite loop in the layout engine in the case where a cell within a row had so much data that it could not possibly fit on a single page. If this case occurs, I now just display as much as possible of that row and move on.
  • Newly created character sheets and saved character sheets that you've opened but made no changes to are no longer marked as modified until you actually make a change.

Miscellaneous

  • Added an equipment list for Powerstones, in 1 point increments all the way up to 300 points. The value numbers exactly match those on M20, and the ones that aren't listed on that page were generated using their formula for cost, then applying a scaled rounding similar to what the book did.
  • Fixed some typos in some of the data files, added some entries that were missing, and cleaned up some entries.
  • Added some more templates.
  • Added the command key SHIFT-CMD-P to the Page Setup menu item.
  • The duplicate command now tries to insert the duplicate next to the original (previously, it would put it at the end of the list).

  • The Vitals hit location now tracks with the Torso. Previously, it was separate, yet there was no longer any way to armor that spot.

  • The "Equipped" flag, while able to be manipulated while the character sheet was open, was never saved out (it was always set to "yes" on saved equipment). Fixed.

Miscellaneous

  • Fixed a range-checking problem in the new text rendering code that would cause an exception to be thrown during mouse drags outside of the text boundaries.

  • Revised the skill default and free point calculation code to take into account some extra rules in SJG's FAQ about Talents.
  • Allow basic speed to be altered in fractional units via Features.

Miscellaneous

Version 3.0.0.15 — July 10, 2005

Features

  • When creating new items in lists, if there is a single selected item or an item that currently has the keyboard focus (i.e. one of its columns is being edited), then the new item will be added directly below that item. If the item happened to also be an open container, the new item will be added as a child of that container rather than as a sibling. If, on the other hand, there are multiple items selected, then the last one in the selection is used, as above. In the case of no selection at all, then the item is added to the end of the list, as before.
  • Various fields in the upper portion of the character sheet, such as attributes, now show the number of character points spent on them in their tooltip.

Bug Fixes

  • The description randomizer was using the randomized height value when trying to set the new weight value. Now it correctly uses the weight value.
